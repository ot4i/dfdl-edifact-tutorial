Overview

This tutorial demonstrates the modeling of UN/EDIFACT data using DFDL schema. The DFDL schema editor is used to view the model and test parse example EDIFACT data files.

UN/EDIFACT data streams have a hierarchical structure where the top level is referred to as an interchange, and lower levels contain multiple messages which consist of segments, which in turn consist of composites and elements. Composites in turn consist of elements. Segments, composites and elements are delimited by separators. Only segments have initiators.

An interchange starts with a UNB segment and ends with a UNZ segment. A message starts with a UNH segment and ends with a UNT segment, an element in the UNH segment identifying the message type. Messages may be grouped into functional groups, which start with a UNG segment and end with a UNE segment. Optionally the interchange can start with a UNA segment which defines the delimiters used by the interchange; defaults are used if there is no UNA.

UN/EDIFACT data streams adopt one of several syntax levels, the most common being version 3 or version 4.

Import projects

Two shared libraries are imported into your workspace.

Library EDIFACT-D03B contains a pair of DFDL schemas that model a UN/EDIFACT Supply Chain interchange for release D.03B. Message types INVOIC and ORDERS are provided. Test data files are included in the library.

Library EDIFACT-Common contains a DFDL schema to define default values for DFDL properties, and a DFDL schema to model the Uxx service segments and service messages for syntax version 4.

Follow these steps to complete the tutorial

The EDIFACT-D03B and EDIFACT-Common libraries are shown in the Application Development view of your workspace.

  1. In library EDIFACT-D03B, open EDIFACT-SupplyChain-Messages-D.03B.xsd using the DFDL editor. The Interchange message will be highlighted.
  2. Test parse an example data file:
    • Click the Test Parse Model button. The Test Parse Model dialog opens.
    • In the Parser Input section, select radio button 'Content from a data file' and click Browse.
    • Select file ORDERS_D.03B_Interchange.txt from EDIFACT-D03B and click OK.
    • Set the Encoding to ASCII.
    • Click OK and confirm the switch to the DFDL Test perspective.
  3. The DFDL Test perspective opens, and the results of the test parse are displayed. You should see 'Parsing completed successfully' which can be dismissed.
  4. You can view the parsed data file in the DFDL Test - Parse view. The results of the parse can be viewed in the DFDL Test - Logical Instance view, as a tree or as XML. You can view a log of the parser actions in the DFDL Test - Trace view.
  5. Test serialize the logical instance that resulted from the parse:
    • Click the Test Serialize Model button. The Test Serialize Model dialog opens.
    • In the Serialzer Input section, select radio button 'Content from a DFDL Test - Logical Instance'.
    • Set the Encoding to ASCII.
    • Click OK.
  6. The results of the test serialize are displayed. You should see 'Serialization completed successfully' which can be dismissed.
  7. You can view the serialized data file in the DFDL Test - Serialize view. You can view a log of the serializer actions in the DFDL Test - Trace view.
  8. Repeat steps 2 through 7 for another example data file, INVOIC_D.03B_Interchange_with_UNA.txt.

The supplied DFDL schemas will parse UN/EDIFACT interchanges with or without new lines at the end of each segment, but will serialize interchanges without new lines. If you need to serialize with new lines, you can edit EDIFACT-Service-Segments-4.1.xsd in the EDIFACT-Common library and change the DFDL expression used to set DFDL variable 'SegmentTerm' to the commented-out value.

If you need to support UN/EDIFACT syntax version 3 instead of syntax version 4, you can edit IBM_EDI_FORMAT.xsd in the EDIFACT-Common library and change DFDL variable 'RepeatSep' to have default value '+' instead of '*'.

If you need to support ',' (comma) as the default decimal separator instead of '.' (period), you can edit IBM_EDI_FORMAT.xsd in the EDIFACT-Common library and change DFDL variable 'DecimalSep' to have default value ',' instead of '.'.

The two shared libraries can be deployed to an integration node for use by message flows.
